🎯 구현 완료: 쿠키 기반 서버 간 인증 연동
✅ 추가된 기능 (방법 2 방식)

1️⃣ FastAPI 서버 (포트 8000)

새 API 추가: /api/verify-session
Spring Boot가 쿠키를 전달하면 "이 쿠키 유효해요?" 확인해줌
유효하면 {"valid": true, "username": "hong"} 응답
CORS 설정: Spring Boot와 통신 가능하도록 설정

2️⃣ Spring Boot 서버 (포트 8082)

AuthService 생성: FastAPI에 HTTP 요청 보내는 서비스
사용자가 보낸 쿠키를 FastAPI로 그대로 전달
FastAPI 응답에서 username 추출
Controller 수정: PostController, NoticeController
게시글/공지 작성 시 쿠키를 AuthService로 검증
로그인 안 했으면 401 에러 + "로그인하세요" 메시지
로그인 했으면 FastAPI가 준 username으로 작성자 자동 설정
DTO 수정: author 필드 제거 (자동으로 채워지므로)
CORS 설정: 쿠키 전송 허용

3️⃣ 프론트엔드 (index.html)

로그인 버튼 추가: "FastAPI 서버에서 로그인하기" 버튼
작성자 입력 칸 제거: 자동으로 채워지므로 불필요
credentials 설정: fetch에 credentials: 'include' 추가 (쿠키 자동 전송)

🔄 작동 흐름
1. 사용자가 http://localhost:8082 접속
2. "FastAPI 서버에서 로그인하기" 버튼 클릭
3. → http://localhost:8000/login 이동
4. FastAPI에서 로그인 성공 → 쿠키 설정 (username=hong)
5. 다시 http://localhost:8082 돌아옴
6. 게시글 작성 버튼 클릭
7. → Spring Boot가 받은 쿠키를 FastAPI로 전달
8. → FastAPI: "유효해요, hongㅇ니다" 응답
9. → Spring Boot: author="hong"으로 게시글 저장 ✅

🎁 장점
✅ 기존 FastAPI 로그인 시스템 그대로 사용
✅ JWT 같은 복잡한 개념 불필요
✅ 사용자는 그냥 로그인만 하면 됨
✅ 쿠키 httpOnly로 보안 유지
